DEFICIT 01 · DASHBOARD AGGREGATES
Portfolio Value & Total Savings REAL-TIME ✓
Both stats are already computed live from the products array fetched from Supabase on mount. portfolioValue = sum of current_price; totalSavings = sum of positive (original_price − current_price) deltas. No hardcoded integers found.
Active Alerts Count MISSING STAT
The stats grid has 4 tiles: POSITIONS, PORTFOLIO, PRICE DROPS, TOTAL SAVINGS. There is no ACTIVE ALERTS tile. price_alerts is never queried on the Dashboard — the BottomNav badge count lives in StashdApp.jsx but a dashboard tile would close the loop.

Fix: Add a 5th stat pulled from price_alerts count query, or replace the least useful tile (PRICE DROPS overlaps with TOTAL SAVINGS). Recommendation: keep 4 tiles, replace PRICE DROPS with ALERTS ACTIVE using gold accent color.
POSITIONS & PRICE DROPS REAL-TIME ✓

DEFICIT 02 · TREND VISUALIZATION · LIBRARY RECOMMENDATION
Recharts — Already Installed RECOMMENDED ✓
Recharts v2.15.4 is already in package.json and PriceChart.jsx already uses LineChart, Line, ResponsiveContainer, Tooltip, YAxis. The gold palette, dark tooltip, and 30/60/90d toggle are all implemented. Zero install cost.

Why not Victory or Chart.js? Victory adds 80 kB for no gain over Recharts' already-used primitives. Chart.js requires Canvas and manual theming. Recharts is React-native, tree-shakeable, and the chart shell is already production-quality.
Root Cause — Chart shows nothing DATA PIPE BROKEN
ProductDetail.jsx line 15: const priceHistory = product?.priceHistory ?? null
The prop priceHistory is never populated — it's a camelCase key that no Supabase query ever sets. The PriceChart component is gated behind {priceHistory && <PriceChart />}, so it never renders.

Fix: In ProductDetail, useEffect on mount to query price_history table by product.id, ordered checked_at ASC. Map rows to { date: row.checked_at, price: row.price }. Pass to PriceChart. Also seed a history row on every product save (INSERT into price_history with the scrape price) so new products always have at least one data point.

PriceChart fix: Currently maps (price, i) => ({ day: i+1, price }) — loses checked_at dates. Update to use real dates on X-axis with XAxis dataKey="date" and a date formatter tick, or keep day index but display the date in the tooltip.

DEFICIT 04 · IMAGE RESILIENCE & PLACEHOLDER DESIGN
Scraper — image_url Reliability HOTLINK RISK
Root cause: Gemini extracts image_url from raw HTML. For retailers like Amazon, Best Buy, and Walmart, the extracted URL is the CDN image URL — valid when scraped but often blocked for cross-origin display due to Referer checks or signed URL expiry.

Current fallback: All placeholder uses are the 📦 emoji in a dark div. Not brand-aligned.

Fix: Replace 📦 emoji placeholders across Dashboard, ListDetail, Alerts, and ProductDetail with a premium SVG placeholder: dark #141414 background, gold "S" monogram or wordmark, subtle radial glow. Add onError handler on every <img> tag to swap to placeholder on load failure.

BOTTOMNAV · CSS WEIRDNESS ROOT CAUSE
FAB clipping & tap-target overlap 2 ROOT CAUSES
Issue 1 — Backdrop blur / stacking context: When any parent element has transform, filter, backdrop-filter, or will-change, it creates a new stacking context that traps position:absolute children. The FAB is position:absolute inside a position:fixed nav — this works in most cases but the box-shadow: 0 0 0 4px #0D0D0D ring can be clipped by adjacent overflow:hidden on scrollable page containers.

Issue 2 — FAB tap dead zone: The FAB protrudes 28px above the nav bar into the page scroll area. Touches in that 28px zone land on the scroll layer, not the FAB button. Since the nav is position:fixed and the page content has its own scroll container (overflow-y:auto; height:100vh), pointer events from the page layer intercept FAB taps in the protruding region.

Fix: Add pointer-events: none to the page scroll containers' bottom padding area (the pb-40 / pb-32 zones are already there to prevent content hiding). The real fix is ensuring the nav's z-index:40 and FAB's z-index:50 exceed any page scroll container z-index. Also: the overflow: visible on the fixed nav is correct and necessary — confirm no ancestor element is resetting it.
